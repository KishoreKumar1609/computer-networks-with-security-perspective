# IP Addressing and Security

---

## What is an IP Address?

An **IP Address (Internet Protocol Address)** is a unique logical identifier assigned to devices connected to a network.  
It allows devices to **identify and communicate with each other across networks**, including the Internet.

Every device connected to the Internet — such as computers, servers, routers, or smartphones — uses an IP address to send and receive data.

IP addresses operate at the **Network Layer (Layer 3)** of the OSI model.

---

## Types of IP Addresses

There are two major versions of IP addresses:

- **IPv4** – 32-bit address format (example: `192.168.1.1`)
- **IPv6** – 128-bit address format designed to overcome IPv4 address exhaustion

Most traditional networking concepts and security discussions focus on **IPv4**.

---

## IPv4 Address Classes

IPv4 addresses are divided into different classes based on network size.

| Class | Address Range | Default Subnet Mask | Usage |
|------|---------------|---------------------|------|
| A | 1.0.0.0 – 126.255.255.255 | 255.0.0.0 | Very large networks |
| B | 128.0.0.0 – 191.255.255.255 | 255.255.0.0 | Medium-sized networks |
| C | 192.0.0.0 – 223.255.255.255 | 255.255.255.0 | Small networks |
| D | 224.0.0.0 – 239.255.255.255 | - | Multicast |
| E | 240.0.0.0 – 255.255.255.255 | - | Experimental |

---

## Public vs Private IP Addresses

### Public IP Address

A **Public IP Address** is assigned by an **Internet Service Provider (ISP)** and is visible on the Internet.

Characteristics:
- Globally unique
- Used for Internet communication
- Assigned to routers or public-facing servers


  
Public IP addresses allow devices to communicate across the global Internet.  
However, because they are publicly reachable, they also become **targets for various network attacks**.

Organizations therefore use multiple security layers such as firewalls, NAT, and monitoring systems to protect public-facing systems.

---

### Private IP Address

A **Private IP Address** is used within internal networks such as homes, offices, or enterprise environments.

These addresses are **not routable on the public Internet** and are reserved specifically for internal communication.

Private address ranges include:

| Range | Usage |
|------|------|
| 10.0.0.0 – 10.255.255.255 | Large enterprise networks |
| 172.16.0.0 – 172.31.255.255 | Medium-sized private networks |
| 192.168.0.0 – 192.168.255.255 | Home and small office networks |

Private addresses typically communicate with the Internet through a router using **Network Address Translation (NAT)**.

---

## Network Address Translation (NAT)

**NAT** is a technique used by routers to translate private IP addresses into a public IP address when accessing the Internet.

This allows many internal devices to share a single public IP address.

### How NAT Works

1. A device inside the network sends a request to the Internet.
2. The router replaces the private IP with its public IP.
3. The destination server responds to the router.
4. The router forwards the response back to the correct internal device.

### Security Benefits of NAT

- Hides internal IP structure
- Prevents direct access to internal systems
- Reduces external attack surface

However, NAT **is not a complete security solution**, and additional protection mechanisms are required.

---

## IPv6 Overview

**IPv6** was introduced to address the limitations of IPv4, especially the shortage of available IP addresses.

IPv6 uses **128-bit addressing**, allowing a vastly larger number of unique addresses.

Example IPv6 Address: 2001:0db8:85a3:0000:0000:8a2e:0370:7334


Key advantages of IPv6 include:

- Massive address space
- Built-in support for IPsec
- Improved routing efficiency
- Better support for modern Internet infrastructure

Although IPv6 adoption is increasing, many networks still rely heavily on IPv4.

## IP Address Allocation and Management

IP addresses on the Internet are distributed through a hierarchical system.

### Global Allocation Structure

1. **IANA (Internet Assigned Numbers Authority)** manages global IP resources.
2. **Regional Internet Registries (RIRs)** distribute addresses regionally.
3. **Internet Service Providers (ISPs)** assign addresses to organizations and users.

Major Regional Registries include:

| Registry | Region |
|--------|--------|
| ARIN | North America |
| RIPE NCC | Europe |
| APNIC | Asia-Pacific |
| LACNIC | Latin America |
| AFRINIC | Africa |
